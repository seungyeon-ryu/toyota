UI GUIDE

1. HTML

1-1. 기본

  • W3C(World Wide Web Consortium) 명세에 의거하여 웹표준을 준수하여 코딩한다. (https://www.w3.org)
  • 문서 최상단에는 HTML5의 독타입 <!doctype html>을 선언한다.
  • 모든 html파일의 인코딩은 utf-8로 통일하여 설정한다.
  • Internet Explorer Version 보정을 위해 IE-edge를 선언한다.
  • 웹 접근성 연구소 지침 가이드에 의거하여 논리적인 컨텐츠로 구성한다. (http://www.wah.or.kr)

1-3. 네이밍

1-3-1. 규칙

A. 일반
  • 이름은 영문 소문자, 숫자를 조합하여 언더바(_)로 구분하여 작성한다.
  • 이름의 시작은 영문 소문자로 제한한다.
잘못된 예 올바른 예
01_icon_notice icon_notice_01
NoticeIconBtn btn_notice_icon
B. 예약어
  • 예약어가 있는 경우 예약어를 사용한다.
  • 예약어는 공통으로 사용하는 컨텐츠에 근거하여 작성한다.
  • 예약어가 없으면 해당 컨텐츠의 의미에 맞게 작성한다.
C. 조합
  • 영문 소문자, 언더바(_)를 이용하여 작성한다.
  • 특수한 경우에는 유동적으로 하이픈(-)을 사용할 수 있다. (ex. plug-in)
  • 언더바(_)는 2개이상의 단어를 조합할 때 사용한다.
  • 네이밍은 되도록 3개의 단어 이내로 작성한다.
잘못된 예 올바른 예
brandData brand_data
main_notice_list_wrap main_notice_list

1-3-2. ID & CLASS

  • ID값은 고유 식별자 이므로 한페이지에 1개를 초과하여 사용할 수 없다.
  • ID값은 주로 레이아웃을 정의할때와 스크립트 동작을 위해 사용한다.
  • 레이아웃을 정의하는 ID는 단어를 조합하지 않고 1개의 단어로만 사용한다.
  • 레이아웃 컨텐츠 외 다른 내용들은 CLASS로 네이밍한다.
  • 독립적인 컨텐츠 또는 자바스크립트와 연동할 경우 ID값을 우선으로 사용한다.
  • CLASS는 중복으로 여러개 정의가 가능한다.
  • CLASS는 의미_구분_타입_순서 로 작성한다.
잘못된 예 올바른 예
#brand_wrap #brand
.brand01_visual_history .brand_history_visual_01

1-3-3. 이미지

  • 영문 소문자만을 사용하여 작성한다.
  • 같은 분류의 이미지가 두개 이상이면 마지막에 언더바(_)와 숫자를 추가하여 구분한다.
  • 특수문자는 사용할 수 없다.
  • 컨텐츠 이미지인 경우 의미_구분_숫자 로 작성한다.
  • 공통으로 사용되는 이미지 경우 형태_의미_숫자 로 작성한다.
잘못된 예 올바른 예

download_btn.gif btn_download.gif

1-4. 레이아웃

A. 예약어

레이아웃의 예약어는 다음과 같이 사용한다.

A. #wrap
B. #header
C. #gnb
D. #container
E. #aside
F. #contents
G. #footer
예약어 설명
#wrap 페이지 전체
#header 페이지 상단
#container 페이지 중단
#footer 페이지 하단
#main 메인 페이지 본문

B. 조합

  • 레이아웃 네이밍은 숫자, 언더바(_)조합을 사용할 수 없다.
  • 레이아웃 네이밍 또는 ID 네이밍 정의시 의미있는 단어를 작성해야 한다.
  • 하나의 아이디값에 다른 타입의 스타일을 적용해야 할 경우 클래스를 추가하여 컨트롤 한다.

1-5. 기본 컨텐츠

A. 기본규칙

  • 컨텐츠에는 내용과 관련있는 클래스를 부여한다.
  • 여러페이지에서 공통으로 사용하는 컨텐츠에는 재활용성이 좋은 이름을 부여한다.
  • 웹페이지 렌더링 최적화를 위하여 불필요한 이미지 사용을 지양한다.
  • 모든 태그의 속성은 큰따옴표(")로 감싼다
  • 모든 태그의 속성은 태그고유속성, id, class, name 순으로 선언한다.
  • 모든 태그의 이름은 영문 소문자만을 사용한다.
  • 들여쓰기를 이용하여 코드의 가독성을 높인다.
  • 태그와 태그사이에 빈줄은 지양하고 레이아웃과 레이아웃사이에만 한줄의 빈칸을 두어 구분한다.

2. CSS

2-1. 기본

  • CSS2와 CSS3를 기준으로 작성한다.
  • CSS파일 최상단에 반드시 인코딩을 선언한다. 파일을 저장할때도 동일한 인코딩을 선택한다.
  • 폰트명, content의 속성값, filter의 파라미터값은 작은따옴표(')로 감싼다.
  • @charset, @import 속성값은 큰따옴표(")로 감싼다.
  • 프로젝트에 따른 CSS 작성 규칙에 맞게 정의한다. (한줄 타입, 여러줄 타입)
  • 그룹화 된 CSS내용에는 주석으로 설명을 적어놓는다.
  • CSS그룹 사이에 빈줄을 허용한다.
  • hack CSS의 사용은 지양한다.
  • 독립적으로 사용될 CSS는 태그내 inline style로 선언한다.
  • CSS선언 타입은 External type을 기본으로 사용한다.
  • CSS 코드를 작성할 때는 들여쓰기를 하지 않는다.
    잘못된 예 올바른 예
    #content {border: 1px solid #000000;}
    　　#content .class {color: #000000;}     		#content {border: 1px solid #000000;}
    #content .class {color: #000000;}

2-3. 선택자

  • 공통 선택자 *는 페이지 성능을 떨어뜨림으로 사용을 지양한다. (부트스트랩 사용시 예외)
  • 여러 선택자에 같은 속성선언시 쉼표(,)를 사용하며 줄바꿈하여 사용한다.
    잘못된 예 올바른 예
    .btn30, btn40 {...} .btn30,
    .btn40 {...}
  • ID값으로 하위선택자 속성 설정시 ID 하위에 ID를 불러오지 않는다.
    잘못된 예 올바른 예
    #brand #outer #inner #brand .outer .inner
  • first-child와 last-child중 하나를 사용해야 할 경우에는 first-child사용을 우선한다. (crossbrowsing)

2-4. 속성

  • CSS작성시 아래의 순서로 속성을 선언한다.
    1. display, 2. overflow, 3. float, 4. position, 5. box-sizing, 6. width, 7. height, 8. margin, 9. padding, 10. border, 11. background, 12. font, 13. line-height, 14. letter-spacing, 15. text-indent, 16. color, 17. align
  • 모든 속성값은 시계방향 순서로 선언한다. (top, right, bottom, left)
  • CSS의 가독성을 높이기 위해 속성과 속성 사이, 속성과 속성값 사이에 공백 한칸을 두어 작성한다.
    div {display: block; position: relative;}

A. display

  • inline 태그에 box level CSS를 사용할 경우 display속성을 block 또는 inline-block으로 선언한다.

B. float

  • float의 해제는 부모 객체에 overflow: hidden을 선언하는 방법을 기본으로 사용한다.
  • 표현의 방향을 고려하여 display: inline-block, clear: both 등의 방법을 사용한다.
  • float 부모 객체에도 float가 선언되었을 때는 따로 해제를 하지 않아도 된다.

C. position

  • fixed속성을 가진 객체의 하위객체는 fixed속성을 다시 가질 수 없다.

D. margin

  • 컨텐츠 사이 간격 및 정렬시 없어질 수 있는 객체에 margin값을 선언하여 가변에 대응한다.

3. JAVASCRIPT

3-1. 기본

  • jquery 기반으로 작성한다.
  • 스크립트 파일 상단에 'use strict'모드를 선언하여 엄격한 개발문법기준을 준수하여 작업한다.
  • plug-in 파일을 먼저 로드하고 내부 스크립트 파일을 로드한다.
  • 재사용성을 높이고 전역변수를 최소화 하기 위해 네임스페이스 패턴으로 작업한다.
    네임스페이스 설명
    newriver 모든 UI 스크립트를 담고 있는 최상위 함수
  • 함수와 함수는 빈줄로 구분하여 가독성을 높인다.
  • jquery셀렉터 및 string타입 선언 등의 작업시 작은따옴표(')를 사용한다.
  • 함수의 네이밍은 의미_상태 순서로 작성한다.
